# Параметры адаптивности

## 1. `adaptive_alpha` - Скорость обновления лямбд

### Что это?
**`adaptive_alpha`** (по умолчанию `0.01`) - это **скорость обучения** или **шаг обновления** для лямбд.

### Как работает?

Формула обновления лямбды:
```python
new_lambda = current_lambda + alpha * delta_avg
```

### Примеры влияния alpha:

#### alpha = 0.01 (медленное обновление, по умолчанию)
```python
# Итерация 1: lambda = 1.0, delta_avg = +0.1
# new_lambda = 1.0 + 0.01 * 0.1 = 1.001

# Итерация 2: lambda = 1.001, delta_avg = +0.1  
# new_lambda = 1.001 + 0.01 * 0.1 = 1.002

# Результат: Медленное, плавное изменение
```

#### alpha = 0.1 (быстрое обновление)
```python
# Итерация 1: lambda = 1.0, delta_avg = +0.1
# new_lambda = 1.0 + 0.1 * 0.1 = 1.01

# Итерация 2: lambda = 1.01, delta_avg = +0.1
# new_lambda = 1.01 + 0.1 * 0.1 = 1.02

# Результат: Быстрое изменение, может быть нестабильным
```

#### alpha = 0.001 (очень медленное обновление)
```python
# Итерация 1: lambda = 1.0, delta_avg = +0.1
# new_lambda = 1.0 + 0.001 * 0.1 = 1.0001

# Результат: Очень медленное изменение, почти не адаптируется
```

### Рекомендации:
- **alpha = 0.01** (по умолчанию) - хороший баланс между стабильностью и адаптацией
- **alpha = 0.05-0.1** - для быстрой адаптации, но может быть нестабильно
- **alpha = 0.001-0.005** - для медленной, стабильной адаптации

## 2. `delta_values` - Значения улучшения операторов

### Что это?
**`delta_values`** - это **среднее относительное улучшение** (или ухудшение) для каждого оператора за итерацию.

### Формула вычисления:

Для каждого потомка:
```python
delta = (лучший_родитель - потомок) / (лучший_родитель + epsilon)
```

Затем вычисляется среднее для всех потомков оператора:
```python
delta_avg = сумма_всех_delta / количество_потомков
```

### Интерпретация значений:

#### delta > 0 (положительное)
```python
delta = +0.05  # Потомок на 5% лучше родителя
# → Оператор работает ХОРОШО
# → lambda увеличивается
# → Оператор будет использоваться ЧАЩЕ
```

#### delta = 0 (нулевое)
```python
delta = 0.0  # Потомок такой же как родитель
# → Оператор работает НЕЙТРАЛЬНО
# → lambda не меняется (или меняется очень мало)
```

#### delta < 0 (отрицательное)
```python
delta = -0.02  # Потомок на 2% хуже родителя
# → Оператор работает ПЛОХО
# → lambda уменьшается
# → Оператор будет использоваться РЕЖЕ
```

### Пример из реальных данных:

Из JSON файла (итерация 1):
```json
"delta_values": {
  "crossover": -0.0229,      // Кроссовер ухудшил на 2.29%
  "mutation": -0.0138,        // Мутация ухудшила на 1.38%
  "scenario1": -0.0037,       // Сценарий 1 ухудшил на 0.37%
  "scenario2": -0.0002,       // Сценарий 2 почти нейтрален
  "scenario3": -0.1267        // Сценарий 3 сильно ухудшил на 12.67%!
}
```

**Что происходит:**
- Все операторы показали отрицательные дельта (ухудшили решение)
- Сценарий 3 показал худший результат (-12.67%)
- Лямбды уменьшатся для всех операторов
- Сценарий 3 получит наибольшее уменьшение лямбды

### Визуализация связи:

```
delta_values → обновление lambda → изменение количества операций
     ↓                ↓                      ↓
  -0.02    →    lambda - 0.0002    →    меньше операций
  +0.05    →    lambda + 0.0005    →    больше операций
```

## Полный цикл обновления

### Шаг 1: Выполнение операций
```python
# Выполняем кроссовер, мутацию, сценарии
# Для каждого потомка вычисляем:
delta = (parent_cost - child_cost) / (parent_cost + epsilon)
```

### Шаг 2: Вычисление средних дельта
```python
crossover_delta_avg = сумма_всех_delta_кроссовера / количество_кроссоверов
mutation_delta_avg = сумма_всех_delta_мутаций / количество_мутаций
# ... и т.д.
```

### Шаг 3: Обновление лямбд
```python
lambda_crossover = lambda_crossover + alpha * crossover_delta_avg
lambda_mutation = lambda_mutation + alpha * mutation_delta_avg
# ... и т.д.
```

### Шаг 4: Применение на следующей итерации
```python
# На следующей итерации:
ncrossover = base_ncrossover * lambda_crossover  # Используем новую лямбду
nmutation = base_nmutation * lambda_mutation
```

---

## Пример из реального запуска

Итерация 1:
- delta_crossover = -0.0229
- lambda_crossover = 1.0
- alpha = 0.01
- **new_lambda_crossover = 1.0 + 0.01 * (-0.0229) = 0.99977**

Итерация 2:
- delta_crossover = -0.0103
- lambda_crossover = 0.99977
- **new_lambda_crossover = 0.99977 + 0.01 * (-0.0103) = 0.99967**

**Результат**: Лямбда постепенно уменьшается, так как кроссовер показывает отрицательные дельта.

## Параметры в конфигурации

```python
config = AdaptiveAlgorithmConfig(
    adaptive_epsilon=1e-5,      # Малое значение для предотвращения деления на 0
    adaptive_alpha=0.01,        # ⭐ СКОРОСТЬ ОБНОВЛЕНИЯ ЛЯМБД
    adaptive_lambda_min=0.4,   # Минимальная лямбда
    adaptive_lambda_max=1.5,   # Максимальная лямбда
)
```

**adaptive_alpha** - это ключевой параметр, который определяет, насколько быстро алгоритм адаптируется к эффективности операторов.
